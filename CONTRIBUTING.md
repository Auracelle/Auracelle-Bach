## ğŸ¤ How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest features
- Provide detailed descriptions with reproducible examples
- Include system information (Python version, OS, dependencies)

### Code Contributions

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes** with clear, commented code
4. **Test thoroughly** - ensure all features work as expected
5. **Commit with clear messages**: `git commit -m "Add: Description of feature"`
6. **Push to your fork**: `git push origin feature/your-feature-name`
7. **Submit a Pull Request** with detailed description

### Code Standards

- Follow PEP 8 style guidelines for Python code
- Include docstrings for all functions and classes
- Add type hints where appropriate
- Write clear, self-documenting code
- Include comments for complex logic

### Testing

- Test new features thoroughly before submitting
- Ensure compatibility with Python 3.8+
- Verify Streamlit interface functionality
- Check API integrations (if applicable)

## ğŸ“ Commit Message Guidelines

- `Test:` Adding or updating tests
